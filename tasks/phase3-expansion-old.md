# Phase 3: 運動メニュー拡充

## 方針
- 運動メニューの追加は**開発者側（アプリアップデート）で行う**
- ユーザーによるカスタム運動作成・画像アップロードは**実装しない**（データ漏洩リスク最小化）
- 運動データ・イラストは引き続き静的ファイルとしてバンドルに含める

## 1. 運動メニュー拡充（20〜30種類目標）
- [ ] 追加する運動の選定
- [ ] ストレッチ系メニュー追加
- [ ] 疾患別カテゴリ追加（膝OA、腰痛、脳卒中 等）
- [ ] イラスト作成（AI生成、既存と同じスタイル）
- [ ] exercises.ts / illustrations.ts に追加

## 2. フィルタ・検索強化
- [ ] 疾患名での検索対応
- [ ] タグ機能（複数タグで絞り込み）
- [ ] 最近使った運動の表示（ローカル履歴）
- [ ] お気に入り機能（ローカル保存）

## 除外項目（意図的に実装しない）
- ~~ユーザーがカメラ/ギャラリーからイラストをアップロード~~
- ~~ユーザー定義の運動作成フォーム~~
- ~~カスタム運動の管理（編集・削除）~~
- 理由: ユーザーデータの増加によるデータ漏洩リスクを避けるため
