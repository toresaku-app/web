# Phase 4: 共有方法の拡充

## 方針
- **ローカル完結を維持** — 外部サーバーへのデータ送信は行わない
- 共有はPDF/画像ファイルの直接共有のみ（iOS標準の共有シート経由）
- QRコード・Webビュー・外部動画リンクは実装しない（漏洩リスク回避）

## 1. 共有方法の最適化
- [ ] LINE直接共有の最適化（PDFファイル共有）
- [ ] メール添付での共有
- [ ] AirDrop共有の確認

## 2. 印刷レイアウトの選択
- [ ] 1ページ1種目（現行、高齢者向け）
- [ ] 1ページ2種目（コンパクト版）
- [ ] レイアウト切り替えUI

## 3. 画像出力
- [ ] PDFではなく画像（PNG）として保存
- [ ] expo-media-libraryでカメラロールに保存

## 除外項目（意図的に実装しない）
- ~~QRコード生成・読み取り~~ — データがエンコードされ第三者に渡るリスク
- ~~患者用閲覧画面（Webビュー）~~ — 外部サーバーが必要になりローカル完結に反する
- ~~動画リンク対応（YouTube等）~~ — 外部通信が発生しローカル完結に反する
- 理由: ローカルオンリー・データ漏洩リスク最小化のコンセプトを維持するため
