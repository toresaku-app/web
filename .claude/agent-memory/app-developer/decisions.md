## アーキテクチャ・設計

### 疾患タグ（conditions）は不採用
- **検討時**: Phase 8 の型拡張で `conditions?: string[]`（膝 OA、腰痛、脳卒中等）を追加予定
- **却下理由**: SaMD リスク。「膝 OA → この運動」と紐づけるとアプリが疾患に対して運動を推奨していると解釈される
- **代替**: 部位・姿勢・カテゴリの 3 軸フィルタで絞り込む。PT が自分の臨床判断で運動を選ぶ形
- **ユーザーの反応**: 即同意。SaMD リスクを理解している
- **再提案厳禁**

### Category 型から「体幹」を削除
- **発端**: ユーザーが「種類のところに体幹って入ってるのはなぜ？」と質問
- **理由**: 「体幹」は部位フィルタ（下肢/体幹/上肢）にも存在し、種類フィルタとの重複で紛らわしい
- **対応**: 体幹カテゴリの運動 4 種（ドローイン、ダイアゴナル、プランク、トランクローテーション）を「筋トレ」に統一。`Category` 型から「体幹」を削除
- **現在の Category**: `"筋トレ" | "ストレッチ" | "バランス" | "ADL" | "呼吸"`
- **教訓**: 体幹は `bodyPart`（部位）であって `category` ではない

### ドラッグ並べ替えではなく上下ボタン方式
- **調査結果**: react-native-draggable-flatlist は Web 対応不安定、@dnd-kit は RN 非対応、PanResponder 自前実装は ScrollView とのジェスチャー競合が複雑
- **採用**: 上下ボタン（▲▼）方式。先頭/末尾では該当ボタンを opacity-20 で無効化
- ストアの `reorderExercises` は既に存在していたので活用

### フィードバック収集は Google Forms
- **却下案**: mailto — メール管理が煩雑、Web 版でメーラー未設定だと送れない
- **採用理由**: 回答がスプレッドシートに自動蓄積、メール設定不要、無料
- **フォーム項目**: カテゴリ（必須: 新しい運動の提案 / 機能リクエスト / バグ報告 / その他）、内容（必須）、ご職種（任意）
- **フォーム URL**: `https://docs.google.com/forms/d/e/1FAIpQLSdnlPwtqKpPcYBHKTdR4XfThPmxwbd3qjPAj3PTih2LD9LhxQ/viewform`
- **UI 配置**: ヘッダー右上（「ご意見」）+ フッター（「ご意見・ご要望」）の 2 箇所

### 休息時間パラメータ
- デフォルト 10 秒、5 秒刻みで増減
- 0 秒にすると PDF に非表示（条件: `sel.restSeconds > 0`）
- `SelectedExercise` に `restSeconds: number` として追加（optional ではなく required、デフォルト 10）

## 開発フロー

### develop/main ブランチ分離 + Vercel ステージング
- **発端**: ユーザーが「今ってテスト環境と本番環境が一緒になってる？」と質問
- GitHub Pages 単体ではステージング困難。Vercel + GitHub Pages 併用が個人開発でメジャー
- main ブランチ保護: `required_pull_request_reviews`（approving_count: 0）+ `enforce_admins: true` + `allow_force_pushes: false`
- `vercel.json`: baseUrl `/web` の問題を解消するため `/web/*` → ルートへのリライト設定

### PR を勝手にマージしない（最重要ルール）
- **事件**: アクセシビリティ改善+ワークフロー整理の PR #3 で、ユーザーの確認前にマージしてしまった
- **ユーザーの反応**: 「今後絶対やめて」と強く注意された
- **対応**: PR 作成後は必ず「プレビューで確認してください」で止める。ユーザーが「マージして」と明示的に言ってからマージする
- 「よろ」と言われた場合は PR 作成までにとどめ、「マージしますか？」と確認する

## インフラ

### リポジトリは public のまま（当面）
- バックエンドなし・API キーなしで漏洩リスク低い
- GitHub Pages 無料プランは public 必須
- **将来**: カスタムドメイン取得 → Cloudflare Pages or Vercel に移行 → private 化

### LP を `public/lp.html` に移動
- **経緯**: 当初は GitHub Actions 内の heredoc で `lp.html` を生成。Vercel 導入後、プレビューで LP が表示されない問題発覚（Expo ビルドに含まれないため）
- **解決**: LP を `public/lp.html` に移動（Expo が自動で dist/ にコピー）。`vercel.json` に `/web/lp.html` → `/lp.html` のリライト追加
- ワークフロー: 旧 LP 生成コード（364 行）を削除済み。現在は Verify step で存在確認のみ
