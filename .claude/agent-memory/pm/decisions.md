## 雇用に関する決定（2026-04-24）

### 決定：pm 職を追加雇用する
- 3エージェント並列運用で全体像が利用者の頭の中だけにある状態を解消するため
- 発端：3エージェント全員が `plan.md` の陳腐化を指摘
- 利用者問い：「全てを統括して plan.md を管理する役はいらないの？」
- 結論：必要。追加雇用する

### 決定：pm は「呼び出し型」運用とする
- **却下**: 4ターミナル常駐型 → 画面が窮屈になる、情報の鮮度も pm が常に起動している必要はない
- **採用**: 必要な時だけ起動（朝イチ / 作業後 / 週次振り返り / 境界チェック）
- **理由**: pm の仕事は episodic（非連続的）。常駐する価値よりセットアップ・メンテコストのほうが大きい

### 決定：pm は実装も決定もしない「情報整理役」に徹する
- **却下**: pm が各エージェントに直接指示を出す「ディレクター型」
  - **却下理由**: Claude Code にはエージェント間通信の仕組みがない。人間がハブである前提を崩すと全体が破綻する
- **採用**: pm → 利用者 → エージェント、の順で情報を流す「情報整理役」
- **権限**: Read 全域、Write は `plan.md` / `tasks/**` / `docs/*-plan.md` / `docs/*-tasks.md` / `docs/marketing-*.md` のみ

### 決定：Bash は読み取り系のみ
- `git status`, `git log`, `git branch`, `ls`, `cat`, `find` のみ許可
- `git commit`, `git push`, `git checkout -b` 等の状態変更系は禁止
- **理由**: 状態変更は app-developer の担当。pm が git を触ると責任が混線する
