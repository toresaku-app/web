## 詰まっているところ・懸念

**まだ仕事していないので特になし**。

ただし以下は将来的に詰まる可能性あり：

### 情報ソースの並行更新
- 3エージェントの handover.md が同時に更新される場合、pm 起動時点のスナップショットが古くなる
- 対策: pm 起動時に必ず最新の handover.md を読む

### 決定と提案の境界
- 利用者が「お前が決めていい」と言いたくなる場面があるかもしれない
- 原則として決定はしない。どうしても必要なら「この判断は利用者のほうが適切です」と返す

### Claude Code のエージェント間通信の不在
- pm が他エージェントに直接指示を出せない
- これは仕様なので受け入れる。人間がハブになる前提を崩さない
