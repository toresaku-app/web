## プロンプトスタイル

### 採用: 元の `PROMPT_TEMPLATE.md` スタイル
```
- セミリアリスティックなフラットイラスト
- 白背景、影なし
- 人物: 中年男性、短髪黒髪、白い半袖Tシャツ、紺のハーフパンツ、白い靴下
- 輪郭線: 細い黒線
- 色数: 最小限（肌色、白、紺、青）
```

### 却下: 強化版スタイル
- 「完全にフラットなベクターイラスト風。グラデーション禁止、影禁止、立体感禁止」
- gpt-image-1 では強化版が良かったが、gpt-image-1.5 では元プロンプトの方が既存イラストとの統一感が高い
- **教訓**: モデルが変わるとプロンプトの効き方も変わる

## 画像生成の運用方式

### フェーズ 1: API 自動生成（Batch 3 前半 + Batch 4 前半）
- Python スクリプト（`scripts/generate-illustrations.py`）で一括生成
- $5 チャージで約 24 枚生成したところでクレジット切れ

### フェーズ 2: ChatGPT 手動生成（Batch 4 後半〜Batch 7）
- Excel プロンプトシートを作成してユーザーに渡す
- ユーザーが ChatGPT にコピペして 1 枚ずつ生成
- 生成画像を `originals/` に入れてもらい、curator がリネーム・WebP 変換・データ登録

### 却下: ChatGPT/Claude サブスク内での自動化
- 調査の結果、サブスクには API アクセスが含まれず自動化不可能
- CLI 経由も同様

### 却下: ブラウザ自動化（Selenium 等）
- OpenAI 利用規約違反のため不推奨

## カテゴリ設計

### Category 型に「体幹」は追加しない
- 一度追加したが、既存の体幹系運動（bird-dog, plank, draw-in 等）はすべて `category: "筋トレ"` だった
- アプリの設計方針として、「体幹」は `bodyPart`（部位フィルタ）で分ける
- ユーザー指摘で元に戻した（side-plank, dead-bug, pelvic-tilt を `category: "筋トレ"` に修正）
- **教訓**: 型変更は curator が独断で行わない。app-developer に依頼する

### Category 型に「ADL」「呼吸」は追加済み（Phase 8-1 で対応済み）
- バランス・ADL・呼吸系の運動が増えるため、フィルタで絞り込めるように
- ただし**UI 側（`CATEGORY_FILTERS`）には未反映**。app-developer 領域の TODO

## ファイル管理

### PNG は `originals/` サブフォルダに退避、アプリは WebP のみ使用
- 以前は PNG と WebP が混在して見づらかった
- PNG（9.7MB）→ WebP（1.1MB）のサイズ差
- `originals/` は `.gitignore` に追加済み

### `.env` の管理
- ユーザーが API キーをチャットに貼ってしまった事例 → 即 Revoke 指示
- `.env` に `OPENAI_API_KEY=sk-proj-...` で保存する運用に変更
- `.gitignore` に `.env` を追加（元々 `.env*.local` のみだった）

## 運動データ設計

### 運動の追加はアプリアップデートで対応
- CLAUDE.md のアーキテクチャ方針
- ユーザーによるカスタム運動作成は実装しない

### イラストは `illustrations.ts` のマッピングで管理
- Exercise 型の `illustration` フィールドは使わない

- 臨床使用頻度と既存カバレッジのギャップから選定
- 下肢（足関節系、タオルギャザー等）、上肢（肩すくめ、顎引き）、ストレッチ（内転筋、股関節内旋）をバランスよく追加して 100 種に到達

## 自動化・ワークフロー

### 画像生成スクリプトのデフォルト
- `gpt-image-1.5` / `high`
- 将来のモデル変更は `--model` フラグで対応可能

- 既存ファイルをスキップ可能に

- フルプロンプト（共通スタイル + 個別）をコピペで済む
